② WebSocket Protocol — WSS /live?token=<cognito_id_token>
Connection handshake
1. Client opens WSS connection with JWT as query param
Note: Header auth not available in browser WebSocket API
2. Server verifies JWT via Cognito JWKS before websocket.accept()
JWT expired → close(4001) · email mismatch → close(4003)
3. On auth success: {"type":"connected","message":"Live stream active"}
4. Server sends keep-alive ping every 30s if no events
Server-sent messages
server→clientPriceUpdate every 5s per open trade
type: "PriceUpdate"
data: { trade_id, symbol, current_price, unrealized_pnl, stop_loss_price, take_profit_price, timestamp }
server→clientping every 30s idle
type: "ping" — client should not respond

⑥ Secrets & Configuration Integration
Secrets Manager consumer map
ib-credentials
EC2 ASG UserData only. Read once at instance boot via instance role. Written to /opt/ibgateway/config/ibgateway.env. Never passed to any Lambda. Fields: username, password, account_id, mode (paper|live).
api-key-{stack}
api_authorizer λ only. GetSecretValue on first invocation, cached in global _cached_key variable (Lambda container lifetime). Never logged.
Runtime configuration via DynamoDB ConfigTable
All runtime flags live in DynamoDB ConfigTable, not Lambda env vars.
This allows live changes from the dashboard without redeployment.

trading_enabled — gate checked by signal λ and risk_check λ on every signal
risk_parameters — max_daily_loss_usd, max_position_size, order_cooldown_secs
daily_pnl#date — cumulative P&L counter, updated atomically via ADD expression
notification_prefs — enabled flag + per-event-type toggles

Lambda env vars hold only non-sensitive config:
TRADES_TABLE, CONFIG_TABLE, POSITIONS_TABLE, EVENT_BUS_NAME,
CP_GATEWAY_URL (internal NLB DNS — stable across ASG replacements),
SNS_TOPIC_ARN, TRADE_LIFECYCLE_ARN, MAX_POSITION_SIZE
