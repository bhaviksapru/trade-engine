Authentication Flows
NinjaTrader → Signal Endpoint
1
TLS 1.3 — API Gateway enforces HTTPS, AWS-managed ACM cert. No plaintext traffic possible.
2
API Authorizer λ checks X-API-Key header against secret stored in Secrets Manager. Key never in code.
3
IP Allowlist — Authorizer λ validates sourceIp from requestContext.http.sourceIp against ALLOWED_IP env var (NinjaTrader machine). Rejects any other IP.
4
Both checks must pass. Single failure → 403, signal dropped, no execution started.
Browser → Dashboard
1
Google OAuth 2.0 via Cognito federated identity. User redirected to Google → consent → Cognito receives ID token.
2
Cognito Pre-Token Trigger λ checks email claim against single ALLOWED_GOOGLE_EMAIL. Any other Google account → token denied.
3
Cognito issues JWT (RS256, signed with Cognito's private key). Token contains email + exp.
4
FastAPI verifies JWT on every request using PyJWKClient with Cognito's public JWKS endpoint. Checks: signature, expiry, audience, email claim.
5
WebSocket path passes JWT as query param ?token=… (browser WS API cannot send headers). Token verified before websocket.accept() — connection closed with code 4001/4003 on failure.

Never hardcoded in OrchestratorClient.cs. Throws at startup if unset.
API key is rotatable — update Secrets Manager + Windows env var + redeploy authorizer.
Rotation & Revocation
IB credentials: rotate by updating Secrets Manager secret → EC2 re-bootstrap or restart container.
API key: rotate in Secrets Manager → authorizer λ in-memory cache invalidates on cold start.
Cognito JWT: 1-hour TTL by default. Revoke by deleting user in Cognito User Pool.

Threat Model — STRIDE
Threat
Control
Residual Risk / Notes
Spoofing
Impersonate NinjaTrader signal
API key (Secrets Manager) + IP allowlist enforced by Lambda Authorizer before Signal λ is invoked
Low — attacker needs both the API key AND the NinjaTrader machine IP. Rotate API key to invalidate compromised keys.
Spoofing
Impersonate dashboard user
Google OAuth + Cognito single-email allowlist + RS256 JWT verified on every request
Low — tied to a specific Google account. Revoking Cognito user invalidates all sessions within JWT TTL.
Tampering
Modify trade signals in transit
TLS 1.3 end-to-end (API Gateway terminates). Payload validated by Signal λ before SF execution starts.
Low. Input validation rejects malformed payloads. Step Functions state is immutable once an execution starts.
Repudiation
Deny a trade was placed
Every trade written to DynamoDB with trade_id, timestamps, execution_arn. CloudWatch logs all Lambda invocations. DeletionPolicy:Retain.
Low. Complete audit trail from signal receipt to close. PITR allows point-in-time recovery.
Info Disclosure
Leak IB credentials
Secrets Manager — EC2 reads once at boot via instance role. Never passed to Lambda. Never logged. No hardcoded creds anywhere.
Low-Medium. Residual: compromised EC2 instance could expose in-memory creds. Mitigate: SSM Session Manager for access (no SSH port open).
Info Disclosure
Expose trade P&L / positions
Dashboard requires Google auth + Cognito JWT. DynamoDB accessible only via IAM-scoped Lambda roles. No public read endpoints.
Low. Residual: CloudWatch logs contain P&L values. Consider log filtering or CloudWatch log data protection.
Denial of Service
Signal flood / runaway strategy
IP allowlist (single source). API Gateway throttling. Orchestrator cooldown (5s). Signal λ checks trading_enabled. risk_check position limit. Step Functions concurrency limit.
Low-Medium. A compromised NinjaTrader machine or rogue strategy could exhaust the position limit before cooldown kicks in. In-flight count fix (risk_check) addresses the race window.
Elevation of Privilege
Lambda escapes its IAM role
Per-Lambda IAM roles (least privilege). No wildcard actions. No cross-account access. Lambda VPC — cannot reach internet except via NAT to IB.
Very low. No admin permissions anywhere. EC2 instance role is limited to one Secrets Manager secret. SSM access is console-only.
Position Stuck Open
Infrastructure failure mid-trade
Step Functions persists execution state — survives Lambda failures. Dead Man λ detects stale heartbeat (>15min) and emergency-closes. Portfolio Risk λ closes on loss limit breach.
Low. The chain of safety nets (SF retry → EmergencyClose state → Dead Man λ) covers most failure scenarios. Residual: simultaneous SF + Dead Man failure requires manual intervention.
